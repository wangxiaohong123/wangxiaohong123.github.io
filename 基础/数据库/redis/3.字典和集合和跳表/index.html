redis-字典、集合、跳表

发表于 2020-04-09 | 分类于 数据库

字典（hash）

和java的HashMap一样，在redis中叫字典hash table，都是使用数组加单向链表的数据结构。字典hash table的结构类似下面这样：

其中dict的type里是一堆函数，ht对应dictht的数组，应该固定是2，两个数组dictht轮着用，在扩容和缩容时切换。

计算index

首先计算出key的hash值，然后拿到hash值和对应dictht的sizemask的二进制做&运算得到key所在的index。如果hash冲突会扩展成单向链表。

rehash

扩容时扩容的长度(2^n^) >= (used*2)，比如说现在存储了15个元素，那么就要找到一个最小的数，这个数要是2^n^，并且这个数要>=30，结果就是32。

缩容时缩容的长度(2^n^)>=used，扩容因子少于0.1就要进行缩容。

在扩容和缩容时，并不是一下把所有数据都迁移，查询是优先查找旧的数据，没有再去新的数组中找，然后把操作数据所在旧的数组index上的链表全部迁移到新的链表，rehashidx++，等到全部迁移完了再把rehashidx设置成-1，把ht[0]指向新数组，ht[1]指向旧数组。

跳表（zset）

score set在redis里的功能是最强大的，类似的LSM树、c0树虽然叫树但是实现是跳表。zset的结构类似下面这样：

每个节点除了分数和值还有一些level和一个BW，level最多有32层，level和level之间有一个连线表示两个相同level之间有多少节点。

当插入元素时首先生成一个1到32的随机数作为节点的level数，也就是层高，然后找到对应位置插入进去。

集合（set）

不重复，当集合中都是数字并且长度小于512时，redis使用intset存储数据，intset的编码是指集合中最大元素占多少位，可以存8、16、32、64位的整数，如果超过64位也不会使用intset了，有时候虽然存储的是long类型，所以我们想在set中存储int取出long是不可能的，只能存储一个long，让intset的编码升级，然后集合中所有的数据就都变成long了。
