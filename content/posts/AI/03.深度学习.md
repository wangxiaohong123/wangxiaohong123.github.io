深度学习的本质：特征表示与结果预测，但是结果预测一般都非常简单、通用的，所以深度学习也叫表示学习或者特征学习。

### 知识库和推理（符号主义）

没有推理能力的模型只是信息处理器，有推理能力的模型是问题解决者。

早期搜索引擎靠的都是倒排索引，搜索引擎就是根据用户的关键词找到接近的文档，所以根据文档内容分词->提取token建立索引指向文档就可以根据关键词搜索文档了，然后再把找到的文档进行排序返回，这就是传统的搜索架构：**召回+排序**。

排序最开始用的是TF-IDF，但是这个算法不好，他没有考虑时间的影响、他不知道文档的质量、以及不能根据用户画像精确排序，比如最新的文章应该更靠前，有的用户喜欢看白话，有的用户需要专业术语更多的文档。所以就演变成了监督训练模型的方式，通过用户看过什么+预测是否点击组成数据集来训练一个二分类模型。

#### RAG系统

RAG最开始用于推荐系统：多通道召回结果统一汇总，系统进行去重、打分之后送进排序系统（Ranking），由CTR模型进行最终的打分排序 = 主流推荐系统的架构。多路召回确保稳定、响应快、覆盖广，精排确保转化高、匹配深、选择准。

随着大语言模型的崛起，RAG也用到了这些生成式AI的检索增强。RAG是为了解决模型联网的问题，因为模型上线之后参数时不变的，没办法访问外部数据和具有时效性的数据。有时候看起来实在认真的回答问题，但是可能是虚假的内容、虚假的文献、虚假的数据，也叫**幻觉**，RAG就是将外部可信的实时数据交给大模型，让大模型的回答更确定。

朴素rag流程：

1. 将原属数据源（各种各样的数据）进行预处理：按照语义边界或者固定长度切分成文本块（Chunk），Chunk是RAG检索的基本单位。

2. 使用嵌入模型（比如BGE、OpenAI Embedding、Cohere Embed）将文本块转成向量，存入向量数据库（比如Faiss、Milvus、Weaviate）并建立索引（比如HNSW、IVF）。

3. 用户触发时将用户的输入使用嵌入模型转成向量，在向量数据库中执行向量搜向量的ANN检索找出语义接近的n个文本块注入LLM。

按照语义边界或者固定长度切分其实相当于断章取义，产生知识割裂。所以标准RAG不适合结构化知识的查询场景，需要引入知识图谱，知识图谱有两个能力：知识搜索和推理。

知识库增强RAG流程：

1. 识别所有实体（NER或者大模型），提取名称、类型、描述，输出的格式应该是`"entity"{{tuple_delimiter}}<实体名称>{{tuple_delimiter}}<实体类型>{{tuple_delimiter}}<实体描述>`。

2. 识别实体之间的关系，对每一对关系实体提取源实体、目标实体、关系描述和关系强度，输出的格式应该是`"relationship"{{tuple_delimiter}}<源实体>{{tuple_delimiter}}<目标实体>{{tuple_delimiter}}<关系描述>{{tuple_delimiter}}<关系强度>`。

3. 编制三元组为知识图谱关系，三元组就是原实体+目标实体+关系。

4. 用户触发时提取实体，然后去图数据库中检索相关的知识，然后将检索出来的原始片段交给大模型，让他输出一个摘要。

5. 将摘组装成prompt再次交给大模型。

#### 智能体

智能体由思维链（CoT） + RAG + MCP + 启发式策略组成。

因为LLM的语义理解是很强的，所以用它来实现思维链：把各种各样的问题交给他，LLM进行问题拆解，将问题拆解成若干个可以通过工具回答的子问题，将所有子问题的答案收集齐了之后整理成最终的答案。

RAG充当的角色是长短时记忆 + 知识背景，MCP提供调用搜索引擎、向量数据库搜索等Function Calling功能，这两部分得到的是思维链拆解出的子问题的答案。

现在还有一个问题需要解决，拆成子问题之后解决的子问题有没有优先级，有没有最优解，怎么判断子问题的结果是否正确，这就需要用到A类搜索算法，正常来说寻找问题的最优解可以把所有情况变成一个决策树，通过深度优先或者广度优先遍历来找到，BFS+剪枝要优于DFS。在AI中我们可以定义一个大概方向，类似于剪枝，去掉一定不可能的情况，比如我们找医院的时候知道大概方向，那么我们只会朝着那个方向走，这种经验导航的原理就是通过$f(n) = g(n) + h(n)$平衡已付出代价和未来潜力，也叫**启发式策略**，虽然不能保证最优，但是可以显著减少无效尝试。

但是目前的主流不是CoT，而是ReAct（(Reason + Act)），大致的流程没有变：

1. 调度中枢：ReAct + 启发式搜索 (如 ToT/MCTS)，LLM不是一次性拆解问题，而是在执行完一个子问题后，根据返回的结果动态决定下一步做什么。

2. 感知与动作层:RAG做知识或者记忆，操作现实数据库或者API。

### 强化学习-RL（行为主义）

RL是控制论的范畴，推荐一本书《控制论与科学方法论》，它属于智能控制论，经典控制论的代表是PID。

#### PID

PID的全称是：Proportional（比例）- Integral（积分）- Derivative（微分）。他的核心公式：

$$
u(t) = \underbrace{K_p e(t)}_{\text{比例 P}} + \underbrace{K_i \int_{0}^{t} e(\tau) d\tau}_{\text{积分 I}} + \underbrace{K_d \frac{de(t)}{dt}}_{\text{微分 D}}
$$

以大模型订阅产品控制系统举例，对应模型而言，有的用户疯狂调用，有的用户不经常使用，对于疯狂使用的用户限制次数或者tokens是不友好的，使用OSI作为大模型服务的核心控制的量化指标。OSI并不是实时的，就需要引入PID控制器，PID的输出表示当前用户的使用成本和平台预期的偏差，通过u可以做一些调节策略，比如：

1. 0<u<0.4：全量大模型，用户体验最佳。

2. 0.4<u<0.7：一部分转向轻量模型，有时回答的不理想。

3. 0.7<u<0.9：切换到小模型或者压缩服务，回答质量下降。

4. 0.9<u：关闭插件，禁用长上下文，可以提示配额用尽。

PID需要调参，减少OSI在1左右晃动的次数，最好一次性回到中心点。P过大震荡剧烈；I过大相应变慢，且容易冲过头；D越大系统越敏感；PID应对多目标很难平衡，比如用户的成本+体验+留存一起权衡，所以他不够聪明。

#### 强化学习

强化学习采用Actor-environment架构，核心逻辑是 **智能体 (Agent)** 在 **环境 (Environment)** 中采取 **动作 (Action)**，根据环境给出的 **反馈/奖励 (Reward)** 来优化自己的 **策略 (Policy)**，目标是获得长期的最大累计奖励。

- Agent：智能体，比如在OSI策略模型中，控制模型切换和分流的策略模块，和思维链（CoT） + RAG + MCP + 启发式策略组成的智能体不是一个概念。

- State：当前状态，比如用户负载，OSI值，预算状态。

- Action：动作选择，是不是切换模型，是否使用长上下文等。

- Reward：奖励，用户体验得分 - 成本惩罚。

- Policy：策略函数，状态到动作的映射规则，就是看到什么之后该如何行动。

RL目前应用在工业自动化与机器人、成本控制系统、路径规划系统、RLHF（AI进化，通过输出多个答案，让人类选择更喜欢哪个）等领域。

##### Q-Learning

现在强化学习已经分出了很多学派，但是他们都是以Q-Learning为基础的，Q-Learning相当于把所有状态对应的所有动作都枚举出来维护到表中，每次都去查表找到最合适的动作执行，公式：现在的得分 = 刚才的旧分 + 学习率 $\times$ (刚拿到的奖励 + 对未来的期望值 - 刚才的旧分)。

Q-Learning非常容易出现表格爆炸，并且输出行为只能是离散的，比如踩油门，他只能记录踩和不踩，或者踩50%之类的，但是没办法输出一个不存在的32.5%。

为了解决Q-Learning的表格爆炸问题，有两种解决方案：

第一种引入了神经网络DQN（Deep Q-Network），本质还是Q-Learning，只是使用神经网络代替表格。但是DQN输出也是离散的，除非把输出层变成1000个神经元，可以控制0.1到100的1000个挡位，这样还是会空间爆炸，模型不好收敛。

第二种是思路是策略梯度 (Policy Gradient)，他不算分，直接学习在A状态下执行动作B的概率是多少，这样可以解决输出离散问题，但也是收敛非常慢，极其不稳定。

目前的主流是演员-评论家派 (Actor-Critic)，结合了DQN和策略梯度，Actor负责直接输出动作，可以是精细的连续数值，Critic负责打分，用来判断Actor输出的好不好。代表算法是PPO和SAC。

### 浅层网络（联结主义）

浅层网络就是只有1层或者几层的网络，最简单的就是1个输入层1个隐藏层1个输出层。

#### 协同过滤

协同过滤可以看作是一个浅层的网络，主要用在推荐算法上，实现物以群分的效果，思想就是喜欢过相同东西的人未来也可能喜欢另一个东西。只有输入层+矩阵分解层+输出层。**矩阵分解为什么能实现协同过滤的效果？？？**

#### 自动交叉

手工交叉是将两个特征组合形成新特征达到1+1大于2的效果，但是手工需要对业务非常了解，运维成本也很高，所以早期facebook发明的GBDT+LR实现自动交叉非常火。GBDT就是几个决策树，LR是逻辑回归，这俩组合到一起也是一个浅层网络。

#### 多层感知机（MLP）

由输入层+1个或多个隐藏层+输出层组成，他是神经网络的祖宗，是一种前馈神经网络。每个隐藏层中的每一个神经元，都与前一层的所有神经元相连（**全连接**）。因为是全连接，所以隐藏层数注定不能太多，但是相比于自动交叉方式还是有很多改进：

| 对比维度 | GBDT+LR           | 端到端MLP     |
| ---- | ----------------- | ---------- |
| 建模流程 | 分阶段，特征工程+模型训练     | 一体化        |
| 特征交叉 | 半自动，受限于GBDT的深度和数量 | 全自动，隐士，高效  |
| 优化目标 | 不一致，GBDT和LR目标分离   | 一致         |
| 工程实践 | 依赖经验              | 简洁         |
| 模型潜力 | 上限低，依赖特征工程质量      | 上限高，依赖网络结构 |

当特征是海量ID，比如雪花算法生成的ID，经过独热编码之后的数据维度就是19位的，这一层的参数量就是相当巨大的，然后Embedding就出现了。最早的Embedding是为了解决One-hot的参数爆炸问题，现在Embedding编码还会带上语义，将现实世界中两个物品的关系映射成距离，这是不是说Embedding只能将类别类的特证进行编码。Embedding最好是监督方式训练，这样才能实现映射。

MLP有一个致命缺陷：记性太差，特征经过层层压缩转换之后有可能丢掉重要的特征，之后Google又提出了Wide & Deep架构，Wide就是负责记忆，是一个简单的线性模型，直接处理原始特征和交叉；Deep就是一个MLP。但是Wide & Deep也有缺陷：如何有效的学习特征之间的复杂关系，依赖耗时的人工设计比如wide部分还是完全交给黑盒的DNN网络；使用共享的Embedding层解决wide和deep的编码不一致问题。所以又出现了**DeepFM / xDeepFM**。

### 深度神经网络（DNN）

前向传播算Loss，反向传播算梯度。

如果多层感知机中有多个隐藏层，那么这个感知机就是一个DNN，但是只有全连接的DNN才是多层感知机，DNN目前除了全连接还有几种类别：

**借口任务/代理任务**：一般自监督的模型训练完很难达到理想效果，这个时候会设置一些任务帮助模型达到目标效果，比如Transformer中的根据上下文猜词、CNN中把图片打碎让魔心还原、RNN的预测上下文词等等。

**Embedding**：就是将一个高位稀疏数据转成低维向量的过程，说白了就是编码+压缩，得到的向量的语义很固定的不涉及上下文的，但是为了让相近的词距离也更近所以不能做简单的encoding，需要找到合适的位置插入值，所以叫嵌入(Embedding)，和他对应的是Hidden state，指的是隐藏层的输出，Hidden state的语义更动态，不过现在都统称为Embedding了。当使用向量数据库检索或者其他情况比如分类的时候就是计算Embedding的距离。

#### 卷积神经网络（CNN的核心）

人类对视觉任务的处理是逐层抽象的泛化过程？在处理图像的理想情况下底层应该识别像素，捕捉基础、敏感特征；中层结果底层的基础特征组合成更复杂的结构基元，比如轮廓；高层聚合中层，形成有明确语义的抽象概念，比如汽车；

物理世界的特点：物质空间是连续的，实体本身也是连续的，不是离散的，因此边缘是视觉最重要的特征。所以CNN的设计基石是局部相关性，学习过程中聚焦于识别这些局部模式，通过堆叠组合成全局概念。

CNN的瓶颈：

理论上更深的网络拥有更大的函数空间，具备更强的表示能力，效果应该更好，但是到达一定程度的网络层数之后，模型效果急剧下降，大部分是由于梯度消失造成的。

反向传播过程中，梯度通过链式法则逐层传递，多个小于1的梯度相乘会导致传递到浅层网络的信号极其微弱，近乎于0，这样浅层网络的参数就没法更新，所以需要一个更稳得信息和梯度传播机制。

ResNet怎么解决梯度消失问题的，虽然ResNet不能100%解决梯度消失，但是让消失的概率降低了很多，ResNet出来之后才开始真正进入大模型阶段，让大量隐藏层堆叠成为可能。目前的模型包括Transformer都在使用。

#### 循环神经网络（RNN的核心）

卷积网络不适合文本处理，因为像素的局部关联，可以让模型根据当前像素以及这个像素周边的其他像素运算，而文字只跟前后文本有关，而且文本对顺序敏感。文本的处理是状态机的形式，和早期计算机很像，根据前2个词理解第3个词，根据前3个词理解第4个词……。

RNN就是为了解决语言任务中的顺序建模和记忆表达的，记忆就是状态吗？RNN维护状态的向量长度固定，且每步都会更新这个向量，导致他的记忆能力有限，越往后可能越乱。所以长短时记忆网络（LSTM）就出现了，他给RNN加了一个外挂的记忆单元。

卷积网络是监督学习，比如图片分类，都有一个确定的分类，是不是所有卷积网络的训练都是监督学习？循环神经网络没法弄那么多带标签的文本数据集，所以才会使用子监督方式，比如挖词、滑动窗口方式预测后面的词等等，间接的变成有监督。

#### 注意力机制（Transformer的核心）

根据上下文猜词（CBOW）；或者跳字模型（Skip-gram），根据中心词预测上下文；因果语言建模（CLM，根据前面的输出预测下一个词）都是一种文本类子监督模型的借口任务，其中CBOW和Skip-gram任务的效果是差不多的，因为他们的本质都是学习词与词的关系，CLM是GPT的借口任务，bert使用的是CBOW的升级版MLM借口任务，因为Skip-gram预测的内容更多，训练效率没有CBOW快。

不管是GPT还是bert看似是预测token，本质是逼着模型学习语言规则，学习事实知识（常识、缩写等等），学习上下文逻辑，学习抽象。因为为了准确的预测下一个词，模型必须具备：

1. 语法规律：比如词法、句法结构
2. 上下文理解：比如主语、谓语一致，代词指代
3. 事实知识：比如北京是中国首都
4. 因果逻辑

为了实现上面的效果模型需要隐式构建了一个记忆宫殿来支持预测。更显示的记忆建模机制是Encoder-Decoder（Seq2Seq）架构，Encoder和Decoder底层都是LSTM实现的，主要用来记忆，Encoder就是将输入转换成Embedding，Decoder将Embedding的语义信息翻译到目标任务上。这个架构最早是用来做机器翻译的，Decoder将输入语言的Embedding转成目标语言就可以了，而且Encoder是可以公用的。

但是LSTM的记忆力也不大，这就导致用户输入很长的时候在Decoder解码完发现丢掉了很多语义。这就是注意力机制诞生的背景。

代替LSTM的注意力机制相当于在模型中放了一个局部向量数据库，因为不能拿这一层的hidden state去查上面层的hidden state输出是否相关，因为两层的Encoder和Decoder语义空间不同。说白了就是当前的hidden state需要拿到之前语义相似的hidden state参与计算，但是不能直接检索，因为不同层的Encoder语义空间不同，（啥叫语义空间，为啥不同），注意力机制使用q k v保证语义空间相同，最开始的交叉注意力中k是Encoder对输入编码之后的向量，v是输入的原始值，q是Decoder层得到的向量。通过q去k的池子中计算相似度找到相似的v？？？Encoder就相当于往向量数据库中插入数据，Decoder是匹配的过程。

基于注意力机制的RNN？？？编码器必须按顺序处理输入，没法并行。

多头注意力

Transformer：编码器方面使用自注意力机制实现并行计算；解码器方面使用查询驱动的生成模式，摆脱对隐藏状态的依赖。

### 三大学派在LLM上的应用

#### LLM与监督微调（SFT，联接主义）

模型1次推理只能生成1个token，每一步推理的输出都是下一步的输入，这个过程叫自回归。所以推理的成本是很高的。

微调的后训练分高效训练和高效微调

高效训练（Efficient Training）就是冻结所有参数，追加并训练头部模型。

#### LLM与专家系统（符号主义）

大模型相当于上下文的消费者，上下文的生产是依赖专家系统来维护、构建的，本质就是LLM和外部系统的信息交换，那针对这个模型设计的接口是不是应该在各个模型或者agent之间是通用的，要不然就很浪费，所以核心问题是统一交互策略，所以就要让交换过程结构化、规范化。所以符号主义为连接主义建立秩序的开放标准MCP（Model Context Protocol）就出现了，这里的Context就是上下文工程中的Context。

上下文工程的组成：

1. Prompt：AI思考的框架，定义任务的逻辑分区，比如人设、任务指令，确保AI理解意图。
2. Tools：AI与外部世界交互的手脚，允许AI调用外部API或执行特定动作，比如发送邮件或者获取实时数据。
3. Resources：挂载的文件，提供AI所需要的外部数据，确保AI基于最新最全的最全的信息推理，他和tools的区别是资源发生改变要实时通知模型，比如在cursor中项目的目录结构发生改变模型在回答的时候就要按照最新的结构调整。

举个例子：用户输入：分析一下谷歌最近的财报。

1. 通过mcp协议寻找匹配的Prompt模板，得到了一个prompt列表，筛选最合适的，比如拿到的是
   
   [人设]你是一位顶级的金融分析师，风格严谨，注重数据
   [用户]{resource:identity_info}
   
   [历史]{resource:conversation_history}
   
   [工具]可用的工具列表：{tools:tool_list}
   
   [任务]根据以上信息，完成任务：{user_intent}

2. 生成资源需求清单：系统扫描上面模板中的占位符，占位符都使用了显式类型（resource、tools这种），所以能够精确的识别类型和具体名称。

3. 通过mcp获取资源，拼成prompt调用模型，可能会涉及到模型返回外部工具的命令，在使用mcp调用外部工具重新组装prompt再去调用模型。

#### LLM与环境交互（行为主义）

RLHF是强化学习的特例，本质上应该是根据环境对其而不是人类这一个单一的信号对齐。人类反馈只是环境反馈的一种情况。

线上的模型基本上就是：生成回答 -> 对回答进行评估 -> 评估的分数作为奖励信号 -> 调整生成策略，让他生成的分数更高。问题是如何评估，或者说奖励从何而来。

比如用户的点击有用/没用、模型的输出进行内容识别，是否有违规内容等等这种奖励模型（RM）打分，然后使用强化学习算法比如PPO进行调整。
