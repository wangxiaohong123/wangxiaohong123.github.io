CompletionService适合处理批量并行任务/任务数量已知并且择优的情况，他和future不同，future的get会阻塞线程，CompletionService将结果的future放到阻塞队列中，这样线程先执行完可以被现获取，比如获取位置服务同时调用三个地图api，当有一个api返回结果时就结束方法：

```java
// 创建线程池
ThreadPoolExecutor executor = new ThreadPoolExecutor(3, 3, 0L, TimeUnit.SECONDS, new ArrayBlockingQueue<>(10, true));
// 创建CompletionService，如果不指定队列，默认使用LinkedBlockingQueue
CompletionService<Integer> completionService = new ExecutorCompletionService<>(executor, new ArrayBlockingQueue<>(10, true));
// 存放线程执行future
List<Future<Integer>> flist = new ArrayList<>(3);
// 提交第一个线程
flist.add(completionService.submit(()-> {
    Thread.sleep(15);
    System.out.println("API1");
    return 1;
}));
// 提交第二个线程
flist.add(completionService.submit(()-> {
    Thread.sleep(5);
    System.out.println("API2");
    return 2;
}));
// 提交第三个线程
flist.add(completionService.submit(()-> {
    Thread.sleep(10);
    System.out.println("API3");
    return 3;
}));
try{
    for(int i = 1; i<= 3; i++) {
        Future<Integer> future = completionService.take();
        if(future.get() != null){
            System.out.println("获取API" + future.get() + "返回值");
            break;
        }
    }
}catch(InterruptedException e){
    e.printStackTrace();
}catch(InterruptedException e){
    e.printStackTrace();
}catch(ExecutionException e){
    e.printStackTrace();
}finally {
    // 获取一个执行结果后，停止所有任务
    for(Future<Integer> f : flist) {
        f.cancel(true);
    }
}
```
